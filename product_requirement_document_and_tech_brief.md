# 🌌 Product Requirements Document (PRD) & Technical Brief

This document serves as the master specification for **Chronicles of Stellar**. It describes the game's product goals, tech stack, physics rendering, AI NPC dialogue loop, Stellar blockchain architecture, and technical rationale.

---

## 🎯 1. Product Requirements Document (PRD)

### Project Concept & Vision
Chronicles of Stellar is a cinematic 2.5D cyberpunk cinematic platformer. The game fuses high-performance action rendering (Phaser) with state-of-the-art decentralized systems (Stellar) and autonomous AI actors (Gemini API).

### User Experience & Game Loop
1.  **Exploration**: The player navigates a 2.5D side-scrolling world across 7 distinct neon sectors using keyboard arrow keys.
2.  **Interaction**: Standing close to an NPC opens a command terminal chat interface in the split-screen HUD.
3.  **Negotiation**: The player chats freely with the NPC. The NPC's reactions (approve, deny, neutral) are generated by a live Gemini AI model based on the player's reputation and prompt.
4.  **Transaction**: If the NPC approves (e.g., selling a passkey or accepting a gate bribe), it triggers a real-time transaction on the Stellar blockchain.
5.  **State Mutation**: Once confirmed by the ledger, the game unlocks new items, increases player reputation, and triggers Phaser animations to lower security firewalls so the player can walk to the next sector.

### Core Product Features
*   **Decoupled Dual-Engine Screen**: Left-side Phaser canvas viewport for smooth 60 FPS physics; right-side React dashboard for wallet status, character status, and AI terminals.
*   **Asynchronous Eventing**: Fast game loops are preserved by sending long-running Web3 ledger checks and LLM calls over an asynchronous event broker.
*   **Dual-mode Ledger**: Support for simulated testnet accounts funded by Friendbot, alongside standard Freighter Wallet setups.

## ⚙️ 3. Game & Rendering Logic (Phaser)

The rendering framework resides in [CanvasManager.ts](file:///Users/noname/documents/Chronichles-of-Stellar-./src/engine/CanvasManager.ts) and [ParallaxController.ts](file:///Users/noname/documents/Chronichles-of-Stellar-./src/engine/ParallaxController.ts):

*   **Snappy Lateral Kinematics**: Physics velocities snap instantly to zero when keys are released (`setVelocityX(0)`), creating responsive, arcade-style movement.
*   **Vector Fallback System**: To ensure the game runs even if custom textures fail to load, the engine draws HTML5 canvas vectors dynamically to render backgrounds, skyline silhouettes, platforms, and character outlines themed to each sector.
*   **Proximity Calculations**: In the update loop, the engine calculates distances to determine if a player is close enough to interact with an NPC:
    $$\text{Distance} = \sqrt{(x_{\text{player}} - x_{\text{npc}})^2 + (y_{\text{player}} - y_{\text{npc}})^2}$$
    If $\text{Distance} < 100\text{px}$, it triggers the React Terminal UI.
*   **Parallax Scroll Controller**: Moves background layers at varying speeds relative to the camera (`scrollX * coefficient`) to simulate depth.

## 🤖 4. AI Agent NPC Logic

The dialogue engine resides in [AgentBridge.ts](file:///Users/noname/documents/Chronichles-of-Stellar-./src/ai/AgentBridge.ts) and [TerminalLogic.ts](file:///Users/noname/documents/Chronichles-of-Stellar-./src/ai/TerminalLogic.ts):

*   **Prompt Routing**: Generates instructions based on the NPC's profile and the player's reputation.
*   **Structured Outputs**: Gemini is constrained to return a JSON response matching this schema:
    ```json
    {
      "responseText": "In-character dialog response.",
      "decision": "approve" | "deny" | "neutral",
      "mutateState": "unlock_keycard" | "open_firewall" | null
    }
    ```
*   **AI Simulation Fallback**: If the Gemini API key is missing, the code falls back to regex matching (scanning for words like "buy" or "bribe") to simulate the agent's logic.

## 🔗 5. Stellar Blockchain Architecture

The transaction framework resides in [StellarClient.ts](file:///Users/noname/documents/Chronichles-of-Stellar-./src/blockchain/StellarClient.ts) and [WalletConnector.ts](file:///Users/noname/documents/Chronichles-of-Stellar-./src/blockchain/WalletConnector.ts):

```
[Phaser Interaction] ──> [Gemini Approve] ──> [StellarClient Build Tx]
                                                      │
                                                      ▼
[React HUD Update] <── [Horizon Confirmation] <── [Freighter Sign]
```

1.  **Connection**: Connects to the Freighter extension or creates a simulated developer keypair funded with 10,000 Testnet XLM via Friendbot.
2.  **Transaction Envelope**: Uses `TransactionBuilder` to create transaction envelopes.
3.  **Smart Mutation Operations**:
    *   *Testnet (Metadata)*: Uses `Operation.manageData` to write state mutations (e.g. `unlock_keycard`) directly to the account's ledger record.
    *   *Mainnet (Live Trading)*: Uses `Operation.payment` to execute real XLM transfers between players.
4.  **Confirmation Loop**: Submits transactions to the Horizon RPC nodes. When confirmed, it updates the player's balance and emits events to update the UI and open the game's physical barriers.

## 💡 6. Architectural Rationale: Why Are We Getting Into It?

### Why Web3 in Gaming?
Web3 provides digital scarcity and trustless asset ownership. Instead of game items being locked inside a private database, assets are represented directly on the public ledger.

### Why Stellar instead of EVM?
*   **Gas Cost Insulation**: EVM gas spikes make basic gameplay actions too expensive. Stellar's transaction fees cost fractions of a cent, keeping costs low for players.
*   **Fast Finality**: Stellar confirms transactions in 3–5 seconds, compared to the longer wait times on EVM chains.
*   **Security by Design**: Using Stellar's native operations (like `ManageData` and `Payment`) avoids the security vulnerabilities of custom Solidity contracts.

### Why Gemini NPC Agents?
Traditional games use pre-programmed, static dialogue trees. Using LLMs as game actors allows players to interact naturally with NPCs. Integrating these AI choices directly with blockchain transactions lets players change the game world state through creative dialogue and negotiation.
